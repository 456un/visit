При заказе сайта у клиента часто возникает вопрос:
на чём лучше делать сайт — на CMS или на framework?
Попробуем объяснить.

### Что такое CMS

CMS (Content Management System) — это готовая система управления сайтом.
Проще говоря, это "конструктор", в котором уже есть базовый функционал для стандартного сайта

* админ-панель 
* возможность редактирования страниц
* управление меню
* управление формами
* возможность редактировать контент без программиста

Примеры CMS: WordPress, Bitrix, Joomla.

CMS больше всего подходит для простого интернет магазина, корпоративного сайта, блога

Плюсы CMS:
* ниже стоимость разработки
* много готовых решений и модулей
* простота разработки стандартного функционала

Минусы CMS:
* ограничения в логике
* сложнее реализовать нестандартные функции
* наличие платных решений и модулей
* с ростом количества пользователей и возрастания нагрузки на сайт, требует перехода на самописные решения, из-за "торможения" сайта

### Что такое Framework?

Framework — это набор инструментов для разработки сайта "с нуля под задачу".
Здесь нет лишнего функционала — всё создаётся именно под ваш проект.

Примеры framework: Laravel, Symfony (backend), Vue, React (frontend).

Framework подходит для:
* сложных сервисов
* нестандартной логики
* высоких нагрузок
* создание сайтов без лишнего функционала

Плюсы framework:
* максимальная гибкость
* высокая производительность 
* легче масштабировать
* лучше подходит для сложных проектов

Минусы framework:
* выше стоимость разработки
* дольше срок запуска
* нужен разработчик для поддержки
* более высокие требования к разработчику

| Критерий                             | CMS                                                              | Framework                                                            |
|--------------------------------------|------------------------------------------------------------------|----------------------------------------------------------------------|
| **Что это**                          | Готовая система управления сайтом с админкой                     | Набор инструментов для разработки сайта под конкретную задачу        |
| **Подход к разработке**              | Использование готовых решений и модулей                          | Разработка логики с нуля под требования проекта                      |
| **Скорость запуска проекта**         | Высокая                                                          | Ниже                                                                 |
| **Гибкость логики**                  | Ограниченная рамками CMS                                         | Практически не ограничена                                            |
| **Возможность доработок**            | Возможны, но иногда сложно и дорого                              | Любые доработки без ограничений                                      |
| **Админ-панель**                     | Есть сразу                                                       | Разрабатывается отдельно                                             |
| **Удобство редактирования контента** | Среднее, подходит для непрофессионалов                           | Зависит от реализованной админки                                     |
| **Поддержка без программиста**       | Частично возможна                                                | Практически невозможна                                               |
| **Производительность**               | Средняя                                                          | Высокая, оптимизируется под проект                                   |
| **Масштабируемость**                 | Ограниченная                                                     | Высокая                                                              |
| **Безопасность**                     | Зависит от обновлений и плагинов                                 | Контролируется разработчиком                                         |
| **Обновления**                       | Регулярные обновления CMS и модулей (иногда ломают функционал)   | Обновляется по необходимости                                         |
| **Зависимость от сторонних решений** | Высокая (плагины, темы)                                          | Минимальная                                                          |
| **Качество кода**                    | Разное, зависит от модулей и конкретной CMS                      | Контролируемое                                                       |
| **Стоимость разработки**             | Ниже                                                             | Выше                                                                 |
| **Стоимость поддержки**              | Низкая или средняя                                               | Средняя или высокая                                                  |
| **Время поддержки и доработок**      | Может расти со временем                                          | Прогнозируемое                                                       |
| **Риск конфликтов**                  | Возможны конфликты модулей                                       | Минимальные                                                          |
| **SEO-возможности**                  | Хорошие "из коробки"                                             | Реализуются под стратегию                                            |
| **Интеграции (CRM, платежи, API)**   | Ограничены готовыми модулями                                     | Любые интеграции                                                     |
| **Поддержка высокой нагрузки**       | Ограниченная                                                     | Рассчитана на рост                                                   |
| **Контроль над архитектурой**        | Низкий                                                           | Полный                                                               |
| **Подходит для MVP**                 | Да                                                               | Да                                                                   |
| **Подходит для стартапов**           | Иногда                                                           | Да                                                                   |
| **Подходит для сложных сервисов**    | Не подходит                                                      | Подходит                                                             |
| **Зависимость от разработчика**      | Низкая                                                           | Высокая                                                              |
| **Требования к разработчику**        | Средние                                                          | Высокие                                                              |
| **Срок жизни проекта**               | Для статичных проектов, без постоянного роста нагрузки на проект | Подходит для долгосрочных проектов, если нагрузка растет со временем |
